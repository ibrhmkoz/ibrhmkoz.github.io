Two folders can be synced with rsync as above. However, we can automate this through a git hook:

#!/bin/bash
# Pre-commit hook to replace symlinks with actual content

# Path to source repository content - use $HOME instead of ~
SOURCE_PATH="$HOME/Documents/blog/posts/"

# Path to the linked content in this repository
TARGET_PATH="$HOME/Documents/website/content/posts/"

# Create target directory if it doesn't exist
mkdir -p "$TARGET_PATH"

# Check for and remove symlinks within the target directory
find "$TARGET_PATH" -type l -delete

# Sync the content from source to target
rsync -av --delete "$SOURCE_PATH/" "$TARGET_PATH/"

# Add the changes to the staged area (relative to Git repo root)
git add "$(realpath --relative-to=$(git rev-parse --show-toplevel) "$TARGET_PATH")"
